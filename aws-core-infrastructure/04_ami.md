# 1.AMIやスナップショットとは？
スナップショットとは、EC2に接続されている EBSボリューム、つまり仮想ディスクの中身バックアップ。

一方、AMI は、EC2インスタンスを起動・復元するための サーバー全体のテンプレート。

## スナップショット
EBSボリューム<br>
└ OSやファイルが入ったディスク<br>
     ↓<br>
    スナップショット

  
## AMI

EC2インスタンス<br>
├ OS情報<br>
├ 起動設定<br>
├ EBSスナップショット<br>
└ インスタンス作成に必要な情報<br>

# 2.AMIを使用してスナップショットとインスタンスのバックアップを取得する
## EC2インスタンスからAMIを作成する

対象インスタンスを選択

EC2<br>
↓<br>
インスタンス<br>
↓<br>
対象インスタンスを選択<br>
↓<br>
アクション<br>
↓<br>
イメージとテンプレート<br>
↓<br>
イメージを作成<br>

```
イメージ名：backup-bastion-server-20260611
イメージの説明：Bastion server backup before configuration change
```

再起動はチェック推奨　<br>
整合性が取れるから

イメージを作成

## AMIが作成されたら状態確認

スナップショットも確認する

AMIを作成すると、裏側でEBSスナップショットも作成される。

EC2<br>
↓<br>
Elastic Block Store<br>
↓<br>
スナップショット<br>

ここに、AMI作成に紐づくスナップショットが表示される。

## AMIからEC2を復元する

バックアップから復元したい場合は、作成したAMIから新しいEC2を起動。

EC2<br>
↓<br>
AMI<br>
↓<br>
自分のAMI<br>
↓<br>
対象AMIを選択<br>
↓<br>
AMIからインスタンスを起動
