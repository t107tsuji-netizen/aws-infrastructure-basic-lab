# 1.踏み台EC2を作成する
## 各種設定を行う
EC2<br>
↓<br>
インスタンス<br>
↓<br>
インスタンスを起動

名前を設定する
```
名前：security-lab-bastion
```
AMIを選択する
```
AMI：Amazon Linux 2023
アーキテクチャ：64-bit x86
```
インスタンスタイプを選択する
```
 t3.microを選択
```

キーペアを選択する
```
キーペア：my-EC2-Key
```

## ネットワーク設定をする
ネットワーク設定を実行する
```
VPC：my-vpc-01
サブネット：security-lab-public-subnet-1a
パブリックIPの自動割り当て：有効
```
SGは新規作成せず、作成済みのものを選択

```
security-lab-bastion-sg
```

## IAM Roleを指定する

画面下の詳細設定を開く

高度な詳細<br>
↓<br>
IAMインスタンスプロファイル

```
security-lab-ec2-role
```

同じRoleを複数のEC2に使うことはできる



## 設定を確認して起動
インスタンスを起動

起動後、以下を確認
```
インスタンス状態：実行中
ステータスチェック：2/2
パブリックIPv4：あり
IAMロール：security-lab-ec2-role
SG：security-lab-bastion-sg
```

# 2.内部EC2を作成する

次に、Private Subnetに内部EC2を作成

## 各種設定を実行する

EC2<br>
↓<br>
インスタンス<br>
↓<br>
インスタンスを起動

名前を設定する
```
名前：security-lab-private-ec2
```

AMIを選択する
```
AMI：Amazon Linux 2023
```
インスタンスタイプ
```
t3.micro
```
キーペア<br>
```
my-EC2-Key
```

## ネットワーク設定
ネットワーク設定を実行する
```
VPC：既存VPC
サブネット：既存のPrivate Subnet
パブリックIPの自動割り当て：無効
```

SGは作成済みのPrivate用を選ぶ
```
security-lab-private-ec2-sg
```

外部から直接SSHさせないため、Private EC2にはパブリックIPv4を付けない

## IAM Roleを指定する
高度な詳細<br>
↓<br>
IAMインスタンスプロファイル

Bastionと同じRoleを付ける
```
security-lab-ec2-role
```

インスタンスを起動

起動後、以下を確認
```
インスタンス状態：実行中
ステータスチェック：2/2
パブリックIPv4：なし
プライベートIPv4：あり
IAMロール：security-lab-ec2-role
SG：security-lab-private-ec2-sg
```

# 3.Bastion EC2へSSH接続する

## TeraTermで踏み台サーバーにSSH接続

TeraTermは管理者権限で起動

```
ホスト：踏み台サーバーのパブリックIPv4
サービス：SSH
TCPポート：22
SSHバージョン：SSH2
```
OKを押す

初回接続時にセキュリティ警告が出たら、接続を続行


次の認証画面で以下を設定
```
ユーザ名：ec2-user
認証方式：RSA/DSA/ECDSA/ED25519鍵を使う
秘密鍵：C:\Users\test\Downloads\my-EC2-Key.pem
```
OKを押す

その後踏み台サーバー上で、秘密鍵を展開するファイルを作成
```
vi my-EC2-Key.pem
```
ローカルPCにある秘密鍵の中身を貼り付ける。<br>

管理者メモ帳で開くのがおすすめ<br>

すべてのファイルを選択しているか注意


保存後、権限を変更
```
chmod 400　my-EC2-Key.pem
```
Private EC2へSSH
```
ssh -i ./my-EC2-Key.pem ec2-user@☓☓☓☓☓☓☓☓☓☓
```

ログインできれば、SG設計とEC2配置は成功

確認後、Bastion上の秘密鍵は削除
```
rm my-EC2-Key.pem
```

IAM RoleをAWSコンソール上で確認

今回は踏み台サーバーを使った接続方法を用いたが、SSMを用いたやり方もある。
